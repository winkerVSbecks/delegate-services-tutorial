When to use the delegate pattern?

1. When using non-AngularJS based libraries in an AngularJS app, eg: Angular-Snap.js. In this case, you would pass an instance of the object created by the library instead of the controller.

2. When building components that need to be controlled by other components, eg: Ion Nav View. In this case, you would pass an instance of the controller. We have to use a controller instead of a service because you can actually have multiple instances of a controller.

3. When you want to write code that is not dependant on AngularJS and can be re-used elsewhere eg: Ion Scroll. In this case you can build your "Class" in whatever and instantiate it within the delegate-service.
